Rust is a statically typed language. The compiler can usually infer what type we want to use based on the value and how we use it. In case when many types are possible, such as when we converted a String to a numeric type using parse, we must add type annotation, like this:

    println!("u: i8  = {}", u);

    println!("u: u32 = {}", u);
    let f: f32 = "127".parse().expect("Not a number!");
    println!("f: f32 = {}", f);
    let f: f64 = "127".parse().expect("Not a number!");
    println!("f: f64 = {}", f);
}

Scalar Types

A scalar type represents a single value.

  • Integer Types
  • Floating-Point Types
  • The Boolean Type
  • The Character Type

Integer Types

The default integer type is i32.

Length Signed Unsigned
8-bit i8 u8
16-bit i16 u16
32-bit i32 u32
64-bit i64 u64
128-bit i128 u128
arch isize usize

isize and usize types depends on the computer/OS architecure.

All number literals expect the byte literal allow a type suffix, such as 255u8, and _ as a visual separator, such as 1_000.

Number literals Example
Decimal 98_222
Hex 0xff
Octal 0o77
Binary 0b1111_0000
Byte (u8 only) b'A'

Integer Overflow

  • Debug mode - Rust includes checks for integer overflow, which cause program to panic at runtime.
  • Release mode - The checks are not included. Instead, Rust performs two’s complement wrapping. i.e. in case of u8, 256 becomes 0.

Floating-Point Types

f32 and f64. The default type is f64.

The Tuple Type

Tuples have a fixed length once declared, they cannot grow or shrink in size.

Comma-seprated list of values inside parentheses. The types of the values in tuple do not have to be the same, the type annotations is optional.

The Array Type

Collection of multiple values of the same type. Array is fixed length.

The values are comma-separated list inside square brackets.

// [type; size]
let a: [i32; 5] = [1, 2, 3, 4, 5];

// [3; 5] -> [3, 3, 3, 3, 3]
let a = [3; 5];

Access array by index out of boundary will cause panic.
